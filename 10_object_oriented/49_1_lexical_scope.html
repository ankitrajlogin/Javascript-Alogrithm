Understanding Lexical Scoping in JavaScript

Example of Lexical Scoping

      
        function outerFunction() {
          const outerVariable = 'I am from outer scope';

          function innerFunction() {
            console.log(outerVariable); // Accesses outerVariable from outerFunction's scope
          }

          innerFunction();
        }

        outerFunction();

Explanation

Lexical scoping defines how variable names are resolved in nested functions. It determines the scope of a variable based solely on its position within the textual structure (lexicon) of the code.

When a variable is referenced within a function, JavaScript first looks for that variable in the immediate function scope. If it's not found there, it continues to look in the outer function scope, and so on, until it reaches the global scope.

Inner functions (functions defined within other functions) have access to variables declared in their outer (enclosing) function scope.

Benefits of Lexical Scoping

  • Encapsulation: It promotes encapsulation by allowing inner functions to access variables from their outer functions, while preventing external code from accessing those variables directly.
